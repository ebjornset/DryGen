About dry-gen

The dry-gen tool is developed as an open source software project hosted on Github under the MIT licence

Inspiration

Beside the “don’t repeat yourself” (DRY) prinsiple, formulated in the book The Pragmatic Programmer, Mermaid (.js) is the main inspiration for dry-gen. Their documentation formats are so terse and simple that I was able to get my first diagram up and running in a few minutes by just typing plain text in my editor! Knowing a bit about C# reflection I realized that generating Mermaid diagrams from C# assemblies would be a managable task, and thus dry-gen was borned.

Development tools

As an open source tool “we’re standing on the shoulder of giants”, and we heavily rely on the hard work of others volunteers. Drygen would probably never have seen the light of day without these other execellent tools:

  • The main developement tool is .Net, a free, cross-platform, and open source developer platform.
  • Github is used for our code repository, issue tracker, deployment tool and documetation site
  • We use SonarQube and their SAAS offering SonarCloud (free for open source projects) as our static code analysis tool.
  • We use Mend bolt (free for open source projects) as our security alerts and compliance tool.
  • The documentation site is generated at deploy time using the Jekyll engine.
  • As a commandline tool, we rely heavily on the Command Line Parser Library.
  • YamlDotNet is used to implement the --options-file option.
  • The verb csharp-from-json-schema is using NJsonSchema as the internal engine doing all the hard work.
  • We use Nuke as our build tool, and to generate our Github actions.
  • We use xunit.net as our test running framework
  • Fluent Assertions is used to assert the code is doing what we expects.
  • All out tests are written as Behavioral specifications using Specflow.
